Operation Dictionary

Operation Name

Inputs

Outputs

Description
AddQuestion QuestionDB, answer, content, difficulty, htmlAttachment, questionType, timeAllowed QuestionDB Add the given question to the QuestionDB. The variable lastUsed is set to null because the question was just added. The variable id is set to the next availiable integer, starting from 1 for the first question. The htmlAttachment is optional, but all of the other variables are required. As for constraints: the difficulty is an number between 1 and 5, and timeAllowed can be up to a 3 digit number representing minutes.  
AddTest Test, TestDB TestDB Add the given Test to the TestDB. Preconditions: all fields except message and proctor must be not null and Teacher, name must be a unique combination.  
AddUser User, UserDB UserDB Add the given User to the UserDB. Preconditions: all fields except email must be not null and emplid, userType must be a unique combination.  
DeleteQuestion Question, QuestionDB QuestionDB Delete the given question from the QuestionDB. The given question must already be in the QuestionDB. The returned QuestionDB no longer has Question in it.  
DeleteUser UserDB, emplid, userType UserDB Delete the given User from the UserDB. Preconditions: giving just an emplid will remove all instances of the user, giving a userType in addition only deletes that unique combination.  
EditQuestion (overload 1) Question, QuestionDB, Test Test This operation takes in a Question. The Instructor can remove the Question from the Test, or he can search for a particular Question in the QuestionDB. Then the Instructor can add the Question to the Test.  
EditQuestion (overload 2) Question, QuestionDB, answer, content, difficulty, htmlAttachment, questionType, timeAllowed QuestionDB Edit the given question in the QuestionDB. The same things happen as if the user was adding a question, but instead of putting a new question in the QuestionDB the given question is simply changed to match the given inputs. The htmlAttachment is optional, but all of the other variables are required. As for constraints: the difficulty is an number between 1 and 5, and timeAllowed can be up to a 3 digit number representing minutes.  
EditTest Test, TestDB TestDB Looks up test with name and Teacher from Test from the TestDB and changes all fields except for name and Teacher. Preconditions: name and Teacher combination must exist in TestDB.  
EditUser User, UserDB UserDB Looks up user with emplid from User from the UserDB and changes all fields except for userType. Preconditions: emplid of given user must match that of User in UserDB.  
FindQuestion QuestionDB, answer, content, difficulty, htmlAttachment, id, lastUsed, questionType, timeAllowed   Find a question by id, difficulty, timeAllowed, lastUsed, content, answer, htmlAttachment, or questionType. It is possible to search for more than one field at once. However, each type of variable does not have to be input. If the user just wanted to search by id, that is the only variable that needs to be not null. Null variables will be ignored in the searching. Once a possible match is found, it is put into a list of Questions as a result that the user can view.  
GetTest TestDB, User, name Test Returns instance of Test with given name and User from the UserDB. Preconditions: User and name must match that of Test in TestDB.  
GetUser UserDB, emplid, userType User Returns instance of User with given emplid and userType from the UserDB. Preconditions: emplid and userType must match that of User in UserDB.  
addQuestion Question Test This operation takes in a Question object and adds it to the Test  
analyzeScores Option TestReport This operation takes in a group of the same completed tests and generates a test report based on the option(Question, Type, Point Value) the user has specified.  
courseScores   TestReport This operation takes in a group of completed tests from a specific course and generates a test report based of all the tests administered for the course.

gradeTest Test TestReport This operation takes in a completed test and generates a test report.  
studentScores   TestReport This operation takes in a group of completed tests from a specific student and generates a test report based of all the tests completed for the student.  
testPreview     This operation takes in Question objects and displays them to the Instructor
