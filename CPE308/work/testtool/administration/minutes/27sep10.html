Minutes of the Tool Time Team Meeting
3:00 - 3:20 PM, 27 September 2010



Present: All

Absent: None

Excused: None

Question:
Do you want the database to be blank first or start from an existing database?
Customer Response:
We do not need to worry about importing from an existing tool. There will be a data entry task. Possibly have a department shared database for questions. Make sure that this data is secure.

Question:
Do you want a relationship model for the database?
Customer Response:
UML and ER diagrams are basically the same thing, The model is more of a collection of things and the records. Feel free to do a ER model. No implementation will be done, so size up users to determine best modeling technique. Most likely, mySQL will be used for a database, but other options are available.

Customer Discussion on other products:
Randomization is a low priority feature that I want turned on and off. Make one of the categories on the feature matrix “question types”. I want to be able to write a script that will take the code from the test, compile it, run it, and output an integer value for a score. I want options that ask for how a student gives feedback.
Want two modes, one that runs in a room where the instructor’s machine is a master control, and the student’s are slaves, which functions like a fully closed book test. The other mode is open note/take home.
Statistics are a low priority, but the feedback is important. This gives feedback where the student sees each point they got on each question.  Stats are worthy of a category in the taxonomy, but they aren’t necessary for the requirements.
You (addressed at our team) are the domain experts for test taking. Consider what you would want to see for test taking. Needs to be as fair for test taking on a computer as on paper.
Want to be able to have a chunk of writing that isn’t necessarily a question, and the questions come after it. Possibly supply a section for notes to be added at the end of test generation. Bigger things for formatting would be nice to be imported into the test creation.
