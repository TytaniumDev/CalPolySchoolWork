2.7. File/Edit Scenarios

As Test Tool is a web-based UI, there is no real file or edit menus, aside from the ones in the browser. However, there are equivalent actions that the user may perform that are similar to file/edit commands. Most of these commands are, however, not explicit.

File Commands:

New Functionality:

In the professor interface, there are 2 places where "New" functionality exists. In Test Generation, New functionality is equivalent to choosing a type of generation. In the Question Database, New functionality is equivalent to clicking the Add button on the main Question Database UI.

There is no student "New" functionality.

Open Functionality:

There is 1 place where "Open" functionality exists in the professor interface. In the Question Database, Open functionality is equivalent to clicking Edit on the main UI.

In the student interface, Open functionality exists when the student opens an exam or opens test results.

Save Functionality:

In the professor interface, there are 2 main things that can be saved: questions and exams.

Questions are saved when the user presses the OK button on the "Add a Question to the Question Bank" dialog box (See Section 2.3.1).

Exams are saved when the user clicks Save on one of the Test Creation screens (see Section 2.2).

In the student interface, completed exams are saved when the user submits them.

Edit commands:

Any edit commands (Undo, copy, paste, etc) are embedded in the web browser and work exactly as they would in any web-based application (Edit commands are performed on text).
