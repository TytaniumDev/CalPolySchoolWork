2.1. User Interface Overview

Upon invoking the test tool, the test tool displays a screen based on the logged in user's credentials. If the user is an instructor, the test tool will invoke a screen shown in figure 2.1.1.

Figure 2.1.1: Initial screen for instructor.

If the user is a student, the test tool invokes the screen shown in figure 2.1.2.

Figure 2.1.2: Initial screen for student.


The initial display consists of two windows depending on the logged-in user's credentials. If the user is an instructor, the system will present the user with Figure 2.1.1, allowing the instructor to dive deeper into the test tool program. The tool will allow instructors to generate exams for students, add/edit/remove questions from the database, handle a test in a secure environment to students, view test results after the examinations are turned in, and perform student administrative tasks such as allowing only a certain subset of students to take the test that is being deployed.

First, the users credentials are checked to see if they are allowed into the system set by the instructor's "student administration" windows. The student has an option to "Take A Test" and the system will display a practice/official exam based on the instructor's preferences. The student also has the ability to view past results on any exams taken.
